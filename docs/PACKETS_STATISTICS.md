## 5. Процедуры и сценарии

### 5.1. 🔐 Процедура верификации (SMS-канал)

> **Источник:** Файл `1. Состав команд для конфигурирования (OK).xlsx`, листы 1–6  
> **Описание:** Первоначальная настройка УСВ через SMS перед подключением по TCP/IP.

#### Последовательность (6 пакетов):

```
1. GPRS_APN         → УСВ       (46 байт)   Настройки APN
2. COMCONF          ← от УСВ    (33 байт)   Подтверждение приёма APN
3. SERVER_ADDRESS   → УСВ       (55 байт)   Адрес сервера
4. COMCONF          ← от УСВ    (33 байт)   Подтверждение приёма адреса
5. UNIT_ID          → УСВ       (42 байт)   Запрос идентификатора
6. COMCONF          ← от УСВ    (33 байт)   Ответ с UNIT_ID
```

#### Схема обмена:

```
┌──────────┐                          ┌─────┐
│ПЛАТФОРМА │      SMS-КАНАЛ           │ УСВ │
└────┬─────┘                          └──┬──┘
     │ 1. GPRS_APN     → настройки APN  │
     │ 2. COMCONF      ← подтверждение  │
     │ 3. SERVER_ADDR  → адрес сервера  │
     │ 4. COMCONF      ← подтверждение  │
     │ 5. UNIT_ID      → запрос ID      │
     │ 6. COMCONF      ← ответ с ID     │
     │                                  │
     │  ✅ Верификация завершена        │
```

### 5.5. 📦 Сценарий: Обновление ПО (EGTS_FIRMWARE_SERVICE)

> **Источник:** `4. Передача обновлённого ПО (OK).xlsx`  
> **Описание:** Платформа передаёт прошивку на УСВ частями.
>
> **ГОСТ 6.7.4:** Сервис `EGTS_FIRMWARE_SERVICE` — передача конфигурации и обновления ПО.

#### Механизм (ГОСТ 6.7.4.1)

`EGTS_SR_SERVICE_PART_DATA` — данные разбиваются на части (1..65535):

| Поле | Описание |
|------|----------|
| **ID** | Уникальный идентификатор сущности |
| **PN** | Номер текущей части (1..65535) |
| **EPQ** | Ожидаемое число частей (1..65535) |
| **ODH** | Заголовок сущности (только 1-я часть) |
| **OD** | Данные сущности (1..65400 байт) |

**Заголовок ODH:** OA (00=прошивка, 01=конфигурация), MT (00=периферия, 01=УСВ), VER, WOS (CRC16), FN

**Подтверждения (ГОСТ 6.7.4.3):**
- Промежуточные части → `EGTS_PC_IN_PROGRESS`
- Последняя часть → `EGTS_PC_OK`

> **Примечание:** Для небольших обновлений используется `EGTS_SR_SERVICE_FULL_DATA` (одним пакетом).

#### Этап 1: Аутентификация (TCP/IP) — листы 1–6

```
1. TERM_IDENTITY      → от УСВ     (59 байт)   Идентификация
2. RECORD_RESPONSE    ← от Платф.  (29 байт)   Подтверждение
3. VEHICLE_DATA       → от УСВ     (48 байт)   Данные ТС
4. RECORD_RESPONSE    ← от Платф.  (29 байт)   Подтверждение
5. RESULT_CODE        ← от Платф.  (24 байт)   Результат
6. RECORD_RESPONSE    → от УСВ     (29 байт)   Подтверждение
```

#### Этап 2: Часть 1 прошивки — листы 7–8

```
7. SERVICE_PART_DATA  → от Платф.  (51 236 байт)  Прошивка (часть 1 из 2)
8. RECORD_RESPONSE    ← от УСВ     (29 байт)      EGTS_PC_IN_PROGRESS
```

#### Этап 3: Часть 2 прошивки — листы 9–10

```
 9. SERVICE_PART_DATA  → от Платф.  (29 312 байт)  Прошивка (часть 2 из 2)
10. RECORD_RESPONSE    ← от УСВ     (29 байт)      EGTS_PC_OK
```

#### Схема обмена:

```
┌──────────┐                                    ┌─────┐
│ПЛАТФОРМА │                                    │ УСВ │
└────┬─────┘                                    └──┬──┘
     │  ЭТАП 1: АУТЕНТИФИКАЦИЯ (TCP/IP)           │
     │ 1-6. TERM_IDENTITY → RESPONSE → ...        │
     │                                            │
     │  ЭТАП 2: ПРОШИВКА ЧАСТЬ 1                  │
     │ 7. SERVICE_PART_DATA → 51 236 байт         │
     │ 8. RESPONSE         ← IN_PROGRESS          │
     │                                            │
     │  ЭТАП 3: ПРОШИВКА ЧАСТЬ 2                  │
     │ 9. SERVICE_PART_DATA → 29 312 байт         │
     │10. RESPONSE         ← OK (завершено)       │
```

## 8. Ключевые выводы

1. **Верификация ≠ Аутентификация** — верификация через SMS (настройка), аутентификация по TCP/IP (авторизация по ГОСТ 6.7.2)
2. **Все 5 тестов начинаются с аутентификации** (кроме файла 1, где есть верификация)
3. **SMS используется только в файлах 1-3** — верификация и запросы данных
4. **RECORD_RESPONSE** — самый частый пакет (16 раз, 29 байт) — подтверждение записи
5. **SERVICE_PART_DATA** — самые большие пакеты (51 КБ + 29 КБ) — обновление ПО
6. **Частей обновления ПО может быть от 1 до 65535** (ГОСТ 6.7.4.1)
7. **Направления пакетов согласованы** на основе анализа Excel-файлов и ГОСТ
8. **15 уникальных типов пакетов**, 45 уникальных HEX-значений, 51 общий пакет
