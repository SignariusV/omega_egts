# Руководство пользователя OMEGA_EGTS GUI

## 1. Запуск

### Установка зависимостей
```bash
pip install -e ".[gui]"
```

### Запуск приложения
```bash
omega-egts-gui
# или напрямую
python gui/main.py
```

## 2. Обзор интерфейса

Приложение состоит из:
- **Боковое меню**: навигация между вкладками
- **Рабочая область**: содержимое выбранной вкладки
- **Строка состояния**: статус сервера, количество подключений, пакетов, время

### Вкладки:
1. **🔌 Подключение** — управление TCP-сервером и CMW-500
2. **📝 Сценарии** — выполнение сценариев (основное окно)
3. **✏️ Редактор** — создание и редактирование JSON-сценариев
4. **📋 Логи** — просмотр системных логов
5. **💾 Экспорт** — выгрузка данных

## 3. Настройка

### Настройки приложения
Доступны через кнопку ⚙ в верхней панели:
- **Тема**: Тёмная / Светлая
- **Язык**: Русский / English
- **Автосохранение**: черновиков сценариев

### Настройки сервера (вкладка "Подключение")
- **Порт TCP**: по умолчанию 3001
- **ГОСТ**: версия 2015 или 2023
- **Уровень логирования**: DEBUG / INFO / WARNING / ERROR
- **CMW-500**: IP-адрес и режим работы

## 4. Работа со сценариями

### Выполнение сценария
1. Откройте вкладку **📝 Сценарии**
2. Выберите сценарий из списка (папка `scenarios/`)
3. Нажмите **▶ ЗАПУСТИТЬ СЦЕНАРИЙ**
4. Наблюдайте за прогрессом и статусом шагов

### Типы шагов
- **expect** — ожидание пакета от УСВ
- **send** — отправка пакета УСВ
- **delay** — пауза (задержка)

### Просмотр пакетов
- Таблица пакетов обновляется в реальном времени
- Двойной клик на пакете → открытие детальной информации
- Фильтры: по направлению (RX/TX), по тексту

## 5. Редактор сценариев

### Создание сценария
1. Откройте вкладку **✏️ Редактор**
2. Нажмите **➕ Создать** или выберите шаблон
3. Отредактируйте JSON (подсветка синтаксиса включена)
4. Нажмите **✅ Проверить** для валидации
5. Нажмите **💾 Сохранить** для сохранения

### Структура сценария
```json
{
    "name": "Название сценария",
    "version": "1",
    "timeout": 60,
    "steps": [
        {
            "name": "Шаг 1",
            "type": "expect",
            "match": {"service": 1}
        },
        {
            "name": "Шаг 2",
            "type": "send",
            "data": "HEX_данные"
        }
    ]
}
```

## 6. Экспорт данных

### Выгрузка
1. Откройте вкладку **💾 Экспорт**
2. Выберите тип данных (пакеты, подключения, сценарии, логи)
3. Выберите формат (CSV или JSON)
4. Укажите файл для сохранения
5. Нажмите **▶ ЭКСПОРТИРОВАТЬ**

### Параметры экспорта
- ✅ Включить временные метки
- ✅ Включить ошибки
- ✅ Включить parsed данные
- □ Разделить по типам пакетов
- □ Сжимать результат (ZIP)

## 7. Просмотр логов

### Фильтрация
- По уровню: ALL / DEBUG / INFO / WARNING / ERROR
- По тексту: ввод поискового запроса
- Автопрокрутка: включена по умолчанию

### Действия
- **🗑 Очистить** — очистка отображения (не файла)
- **💾 Экспорт** — сохранение отфильтрованных логов в файл

## 8. Диалоги

### Детали пакета
- Открывается двойным кликом по пакету
- Показывает: время, направление, размер, CRC, заголовок EGTS, записи, HEX-дамп
- Кнопки: **📋 Копировать HEX**, **💾 Сохранить**

### Настройки
- Доступны через ⚙ в верхней панели
- Сохраняются между сессиями (QSettings)

### О программе
- Доступно через ❓ в верхней панели
- Версия приложения, описание, разработчик

## 10. Устранение неполадок

### Сервер не запускается
- Проверьте, что порт не занят другим процессом
- Проверьте права доступа (Windows: запуск от администратора)

### CMW-500 не подключается
- Проверьте IP-адрес и сетевое подключение
- Убедитесь, что режим работы выбран верно
- Проверьте настройки SCPI

### Сценарий не выполняется
- Проверьте валидность JSON (кнопка ✅ Проверить)
- Убедитесь, что УСВ подключено к серверу
- Проверьте логи на наличие ошибок
