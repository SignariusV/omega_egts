# Обновление ПО (EGTS_FIRMWARE_SERVICE)

## Описание
Платформа передаёт прошивку на УСВ частями через EGTS_FIRMWARE_SERVICE (ГОСТ 6.7.4).

## Механизм (ГОСТ 6.7.4.1)

`EGTS_SR_SERVICE_PART_DATA` — данные разбиваются на части:
- **ID** — уникальный идентификатор сущности
- **PN** — номер текущей части (1..65535)
- **EPQ** — ожидаемое число частей (1..65535)
- **ODH** — заголовок сущности (только 1-я часть)
- **OD** — данные сущности (1..65400 байт)

**Подтверждения (ГОСТ 6.7.4.3):**
- Промежуточные части → `EGTS_PC_IN_PROGRESS` (1)
- Последняя часть → `EGTS_PC_OK` (0)

> **Примечание:** В данном тесте 2 части, но по ГОСТ частей может быть от 1 до 65535.

```
│ПЛАТФОРМА │                                    │ УСВ │

     │  ЭТАП 1: ПРОШИВКА ЧАСТЬ 1                  │
     │ 1. SERVICE_PART_DATA → 51 236 байт         │
     │ 2. RESPONSE         ← IN_PROGRESS (1)      │
     │                                            │
     │  ЭТАП 2: ПРОШИВКА ЧАСТЬ 2                  │
     │ 3. SERVICE_PART_DATA → 29 312 байт         │
     │ 4. RESPONSE         ← OK (0)               │
     │                                            │
     │  ✅ ПО обновлено (80 548 байт, 2 части)    │
```
