# Передача траектории движения

## Описание
Платформа запрашивает траекторию через SMS, УСВ передаёт данные по TCP/IP.

## Схема обмена

```
┌──────────┐                                    ┌─────┐
│ПЛАТФОРМА │                                    │ УСВ │
└────┬─────┘                                    └──┬──┘
     │  ЭТАП 1: SMS-ЗАПРОС                        │
     │ 1. TRACK_DATA   → запрос (38 байт)         │
     │ 2. COMCONF      ← подтверждение (33 байт)  │
     │                                            │
     │  ЭТАП 2: ДАННЫЕ (TCP/IP)                   │
     │ 3. SR_TRACK_DATA ← от УСВ (880 байт)       │
     │ 4. RESPONSE      → подтверждение (29 байт) │
     │                                            │
     │  ✅ Траектория получена (71 точка)         │
```

## Шаги

| # | Пакет | Канал | Направление | Размер | Описание |
|---|-------|-------|-------------|--------|----------|
| 1 | EGTS_TRACK_DATA | SMS | Платформа → УСВ | 38 байт | Запрос траектории |
| 2 | CT_COMCONF | SMS | УСВ → Платформа | 33 байт | Подтверждение SMS |
| 3 | EGTS_SR_TRACK_DATA | TCP/IP | УСВ → Платформа | 880 байт | Данные траектории (71 точка) |
| 4 | EGTS_SR_RECORD_RESPONSE | TCP/IP | Платформа → УСВ | 29 байт | Подтверждение приёма |

## Пакеты

- `packets/platform/track_data_request.hex` — SMS-запрос
- `packets/usv/comconf_track.hex` — SMS-подтверждение
- `packets/usv/sr_track_data.hex` — данные траектории
- `packets/platform/record_response_track.hex` — подтверждение
