# Передача профиля ускорения

## Описание
Платформа запрашивает профиль ускорения через SMS, УСВ передаёт данные по TCP/IP.

## Схема обмена

```
┌──────────┐                                    ┌─────┐
│ПЛАТФОРМА │                                    │ УСВ │
└────┬─────┘                                    └──┬──┘
     │  ЭТАП 1: SMS-ЗАПРОС                        │
     │ 1. ACCEL_DATA   → запрос (38 байт)         │
     │ 2. COMCONF      ← подтверждение (33 байт)  │
     │                                            │
     │  ЭТАП 2: ДАННЫЕ (TCP/IP)                   │
     │ 3. SR_ACCEL_DATA ← от УСВ (3 244 байт)     │
     │ 4. RESPONSE      → подтверждение (29 байт) │
     │                                            │
     │  ✅ Профиль ускорения получен              │
```

## Шаги

| # | Пакет | Канал | Направление | Размер | Описание |
|---|-------|-------|-------------|--------|----------|
| 1 | EGTS_ACCEL_DATA | SMS | Платформа → УСВ | 38 байт | Запрос профиля ускорения |
| 2 | CT_COMCONF | SMS | УСВ → Платформа | 33 байт | Подтверждение SMS |
| 3 | EGTS_SR_ACCEL_DATA | TCP/IP | УСВ → Платформа | 3 244 байт | Профиль ускорения |
| 4 | EGTS_SR_RECORD_RESPONSE | TCP/IP | Платформа → УСВ | 29 байт | Подтверждение приёма |

## Пакеты

- `packets/platform/accel_data_request.hex` — SMS-запрос
- `packets/usv/comconf_accel.hex` — SMS-подтверждение
- `packets/usv/sr_accel_data.hex` — профиль ускорения
- `packets/platform/record_response_accel.hex` — подтверждение
